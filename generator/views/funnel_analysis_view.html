generator.views.funnel_analysis_view

Class to describe a Funnel Analysis View.

We create a single View file and a single Explore file.

The View file has many Looker views defined within it: funnel_analysis: Based on events_daily, has the events string and user dimensions (e.g. country) event_names: The names of events. Used for suggestions. event_N: For each possible funnel step, a single view. This is used to define what that funnel step is.

The Explore's job is to take this generated file an link all those event_N's to the funnel_analysis. We join them via cross join, because event_N views only have 1 row and 1 column - the match_string to use for a regex_match on the events string in funnel_analysis.

For example, say we filter event_1 on event: WHERE event in ("session-start, "session-end") Then we join that with funnel_analysis: FROM funnel_analysis CROSS JOIN event_1 That lets us find out whether the user completed those funnel steps: SELECT REGEXP_CONTAINS(funnel_analysis.events, event_1.match_string) AS completed_step_1

The funnel_analysis view has some nice dimensions to hide these details from the end user, e.g. completed_funnel_step_N. We can then count those users across dimensions.

 38        """Get an instance of a FunnelAnalysisView."""
 39        super().__init__(namespace, "funnel_analysis", FunnelAnalysisView.type, tables)

 41    @classmethod
 42    def from_db_views(
 43        klass,
 44        namespace: str,
 45        is_glean: bool,
 46        channels: List[Dict[str, str]],
 47        db_views: dict,
 48        num_funnel_steps: int = DEFAULT_NUM_FUNNEL_STEPS,
 49    ) -> Iterator[FunnelAnalysisView]:
 50        """Get Client Count Views from db views and app variants.
 51
 52        We only create a FunnelAnalysisView if we have the two necessary db tables:
 53            - events_daily
 54            - event_types
 55        """
 56        # We can guarantee there will always be at least one channel,
 57        # because this comes from the associated _get_glean_repos in
 58        # namespaces.py
 59        dataset = next(
 60            (channel for channel in channels if channel.get("channel") == "release"),
 61            channels[0],
 62        )["dataset"]
 63
 64        necessary_views = {"events_daily", "event_types"}
 65        actual_views = {}
 66        for view_id, references in db_views[dataset].items():
 67            if view_id in necessary_views:
 68                actual_views[view_id] = f"`mozdata.{dataset}.{view_id}`"
 69
 70        if len(actual_views) == 2:
 71            # Only create an instance if we have the two necessary tables
 72            tables = {
 73                "funnel_analysis": "events_daily_table",
 74                "event_types": actual_views["event_types"],
 75            }
 76            tables.update(
 77                {f"step_{i}": "event_types" for i in range(1, num_funnel_steps + 1)}
 78            )
 79            yield FunnelAnalysisView(
 80                namespace,
 81                [tables],
 82            )

 85    def from_dict(
 86        klass, namespace: str, name: str, _dict: ViewDict

 88        """Get a FunnalAnalysisView from a dict representation."""

103        dimensions = [

115                            True
116                        )
117                    )

121            for n in range(1, self.n_events() + 1)
